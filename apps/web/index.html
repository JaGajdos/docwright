            Market fit: all of these (except CodeRabbit) offer no-login access
            for public repos and generate some form of diagram/docs with an LLM
            over repo content — Docwright follows that proven pattern.
          
    
          
    
            Where Docwright stands out: you can verify that badges (license /
            version / CI) are real GitHub data, not model guesses — others
            either skip this or keep it opaque. Docwright also offers an
            explicit output-language choice and a readable pipeline codebase.
            DeepWiki has far broader reach (50k+ indexed repos, code chat, MCP)
            — Docwright is intentionally narrower: one README + one diagram on
            demand, not a hosted wiki platform.
